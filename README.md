# Автоматизация тестирования формы авторизации DemoQA

# О проекте

Проект автоматизирует следующие сценарии:
1. **Негативный тест:** Проверка отображения ошибки при вводе неверных учетных данных.
2. **Позитивный тест:** Успешная авторизация, проверка перехода в личный кабинет и корректный выход из системы.

# Стек технологий

*   Python 3.13+
*   Pytest — фреймворк для запуска и организации тестов.
*   Selenium WebDriver — управление браузером Chrome.
*   Allure Report — генерация детальных отчетов о прохождении тестов.
*   python-dotenv — управление конфигурацией и секретными данными.
*   Page Object Model (POM) — архитектурный паттерн для разделения логики тестов и структуры страниц.

# Клонирование репозитория

git clone https://github.com/KarlFutli/test_auto.git
cd test_auto

# Установка зависимостей

pip install -r requirements.txt

# Настройка конфигурации

Создайте файл .env в корне проекта на основе шаблона .env.example и заполните его своими данными:

BASE_URL=https://demoqa.com
VALID_USER=ваш_логин
VALID_PASSWORD=ваш_пароль
INVALID_USER=неверный_логин
INVALID_PASSWORD=неверный_пароль

# Запуск тестов

pytest -v
